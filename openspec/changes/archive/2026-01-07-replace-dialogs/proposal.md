# Proposal: Replace System Dialogs with Custom UI

## Problem

Текущая реализация использует системные диалоги `alert()` и `confirm()`, которые имеют несколько недостатков:

1. **Непоследовательный UX**: Системные диалоги не соответствуют дизайн-системе приложения (кастомные темы, цвета).
2. **Блокирующее поведение**: Системные модальные окна блокируют весь UI и прерывают работу пользователя.
3. **Ограниченная кастомизация**: Невозможно стилизовать системные диалоги или адаптировать их под темную/светлую тему.
4. **Проблемы с тестированием**: Системные диалоги сложно тестировать программно.

### Current Usages

**alert() (4 places):**

- `SettingsPage.vue:48` — успешное сохранение глобальных настроек
- `SettingsPage.vue:50` — ошибка сохранения настроек
- `HomePage.vue:40` — ошибка при удалении курса
- `HomePage.vue:58` — ошибка при сохранении курса

**confirm() (5 places):**

- `CourseSettingsModal.vue:81` — подтверждение сброса настроек к глобальным
- `HomePage.vue:34` — подтверждение удаления курса
- `CoursePage.vue:117` — подтверждение удаления карточки
- `CoursePage.vue:142` — подтверждение удаления выбранных карточек
- `CoursePage.vue:156` — подтверждение удаления всех карточек курса

## Solution

Заменить системные диалоги на кастомные UI компоненты:

1. **alert() → vue3-toastify**: для уведомлений об успехе/ошибке
2. **confirm() → ConfirmDialog**: кастомный модальный компонент с анимациями

## Benefits

- ✅ Консистентная стилизация в соответствии с дизайн-системой
- ✅ Поддержка светлой и темной темы автоматически
- ✅ Более гибкие и настраиваемые уведомления
- ✅ Улучшенная доступность (ARIA attributes)
- ✅ Программируемые и тестируемые компоненты

## Scope

### In Scope

- Установка и настройка `vue3-toastify` (глобальная конфигурация)
- Создание `ConfirmDialog.vue` компонента с анимациями
- Создание `useConfirm.js` composable для программного использования
- Замена всех 4 `alert()` на `toast.success()` / `toast.error()`
- Замена всех 5 `confirm()` на `useConfirm().confirm()`
- Документирование нового подхода в ui-notifications spec

### Out of Scope

- Миграция на другие библиотеки уведомлений (кроме vue3-toastify)
- Кастомизация иконок toast нотификаций (используем дефолтные)

## Migration Notes

> [!WARNING]  
> Все usages `alert()` и `confirm()` должны быть заменены за один раз, чтобы избежать несогласованного UX.
