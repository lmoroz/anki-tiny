# Proposal: Batch Card Delete

## Overview

Добавить возможность массового удаления карточек в интерфейсе курса. Пользователь сможет выбирать несколько карточек с помощью чекбоксов и удалять их одной операцией. Также будет добавлена кнопка для удаления всех карточек курса одним действием.

## Motivation

**Проблема**: В настоящее время пользователь может удалять карточки только по одной, подтверждая каждое удаление через `confirm()`. Это неудобно при необходимости удалить несколько устаревших или неправильно созданных карточек.

**Решение**: Добавить режим выбора карточек с чекбоксами для массового удаления и дополнительную кнопку для очистки всего курса от карточек.

## Goals

1. Добавить режим выбора карточек с кастомными чекбоксами
2. В режиме выбора карточки не должны переворачиваться при клике
3. Выбранные карточки визуально отличаются (уменьшенный opacity)
4. Кнопка "Удалить выбранные (N)" для удаления отмеченных карточек
5. Кнопка "Удалить все карточки" для полной очистки курса
6. Backend endpoint для массового удаления карточек

## Non-Goals

- Восстановление удалённых карточек (undo)
- Корзина удалённых карточек
- Массовое редактирование карточек

## Success Criteria

- Пользователь может войти в режим выбора карточек
- Пользователь может выбрать несколько карточек чекбоксами
- Выбранные карточки визуально отличаются (opacity)
- Карточки в режиме выбора не переворачиваются при клике
- Пользователь может удалить выбранные карточки одной операцией
- Пользователь может удалить все карточки курса одной кнопкой
- Backend поддерживает массовое удаление через один API запрос

## Out of Scope

- История удалений
- Экспорт карточек перед удалением
- Восстановление из резервной копии

## Related Work

- Спецификация `course-ui` (управление карточками в интерфейсе курса)
- Существующее API `DELETE /api/cards/:id` для удаления одной карточки
- `useCardStore` с методом `deleteCard()` для единичного удаления
